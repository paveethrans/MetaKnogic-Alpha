About this project

From 6.6 million papers to a queryable metabolite‚Äìcancer graph

This project builds a Generalized Knowledge Hypergraph (GKHG) for metabolism and cancer. We ingest millions of full-text articles, score them with a quantitative relevance function, extract multi-entity relations with HypergraphRAG, and expose the result through a graph-aware RAG interface.

Hypergraph workflow

  • 1
    Ingest
    Pull full-text XML/HTML from PMC and metadata from OpenAlex. Align papers across the two sources to get both content and citations.
  • 2
    Filter & refine
    Hard and fuzzy match using a metabolism‚Äìcancer keyword list. Compute relevance scores (see below) and drop low-quality, off-topic papers.
  • 3
    Extract (HypergraphRAG)
    For each chunk, HypergraphRAG extracts: entities (metabolites, cancers, pathways, conditions), hyperedges (multi-entity statements), natural language descriptions, and vector embeddings.
    Each hyperedge = one evidence sentence connecting multiple entities.
  • 4
    Build GKHG
    Construct a bipartite graph: entity nodes ‚Üî hyperedge nodes. Store text, metadata, and embeddings for both sides.
  • 5
    Query & reasoning
    Embed the user query ‚Üí retrieve closest entities ‚Üí follow their incident hyperedges ‚Üí aggregate supporting evidence and send it to the LLM.
  • 6
    Answer construction & validation
    The LLM synthesizes an explanation grounded in retrieved entities and hyperedges. We combine this with manual biological sanity checks and relevance-weighted ranking.

Relevance scoring at scale

Every paper gets a continuous relevance score that combines text match, citation impact, recency, and journal prestige. Each component is transformed into a 0‚Äì1 quantile rank to stay robust to outliers.

Text signal (35%)

We combine section-aware boosting and local keyword density:

\[ T_1 = \mathrm{qrank}_{01}(\text{section\_boost}), \quad T_2 = \mathrm{qrank}_{01}(\text{density}_{/1000}) \] \[ \text{text\_signal} = 0.8\,T_1 + 0.2\,T_2 \]

Citation signal (35%)

We log-transform citation counts before ranking:

\[ C = \log\bigl(1 + \text{cited\_by\_count}\bigr) \] \[ \text{cit\_signal} = \mathrm{qrank}_{01}(C) \]

Recency signal (15%)

Penalize very old work, but clip extreme values:

\[ R_{\text{raw}} = \mathrm{clip}\bigl(\text{pub\_year} - \text{base\_year}, -10, 10\bigr) \] \[ \text{recency\_signal} = \mathrm{qrank}_{01}(R_{\text{raw}}) \]

Journal prestige signal (15%)

Combine multiple journal-level indicators:

\[ J = 0.55\,\mathrm{qrank}_{01}(\text{SJR}) + 0.20\,\mathrm{qrank}_{01}(\text{cites\_doc\_2y}) + 0.15\,\mathrm{qrank}_{01}(\text{citescore}) + 0.10\,\mathrm{qrank}_{01}(\text{hindex}) \] \[ \text{journal\_signal} = \mathrm{qrank}_{01}(J) \]

Final relevance score

\[ \text{final\_score} = 0.35\,\text{text\_signal} + 0.35\,\text{cit\_signal} + 0.15\,\text{recency\_signal} + 0.15\,\text{journal\_signal} \]

Here, \(\mathrm{qrank}_{01}(\cdot)\) maps each metric to its empirical quantile rank in \([0,1]\), ensuring the scores are comparable and robust to extreme outliers.

Why KG + RAG instead of GPT-only?

Large biomedical GPT models are strong readers, but they don‚Äôt retain a stable, queryable memory of millions of papers. The hypergraph acts as that structured memory.

LLM-only (GPT-style)
  • Great at local synthesis, but no explicit global graph.
  • Can hallucinate multi-step mechanisms.
  • Hard to inspect which evidence it used.
  • No direct way to trace metabolite‚Äìgene‚Äìpathway‚Äìcondition chains.
GKHG + HypergraphRAG
  • Entities and hyperedges come from explicit sentences.
  • Multi-way relations (drug‚Äìgene‚Äìpathway‚Äìnutrient‚Äìcondition) are preserved.
  • Every answer can be tied back to specific hyperedges (evidence).
  • Graph structure enforces factual grounding and improves explainability.
